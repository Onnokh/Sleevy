# Keyboard-Driven Web Companion with Command Palette

The Web Companion will evolve from a passive viewing surface into a keyboard-driven desktop client. This adds j/k list navigation, single-key item actions, sequence-based page navigation, and a command palette (Cmd+K) that combines Saved Item search, page navigation, action commands, and URL capture in a flat grouped list. Keyboard state lives in an app-level React context provider at the root layout rather than using DOM focus, because multiple competing focus targets (palette, capture form, context menus) make DOM focus synchronization brittle. The provider owns only the selected index, global shortcuts, and palette state; pages pass their list bounds without leaking query or filter logic upward. TanStack React Hotkeys handles keybinding registration for ecosystem consistency with TanStack Router and Query, accepting the alpha-stage stability trade-off. This supersedes ADR 0001's framing of the web app as a secondary surface — iOS remains the primary mobile client, but the Web Companion is now the primary desktop client with its own keyboard-first interaction model.
